# 🟦 Microsoft Interview Guide — Lead/Principal Engineer

> Focus: Behavioral Patterns, Azure, System Design

---

## Interview Structure
| Round | Focus |
|-------|-------|
| Phone Screen | Coding + Behavioral |
| Loop 1 | System Design |
| Loop 2 | Coding (design-oriented) |
| Loop 3 | Behavioral (Growth Mindset) |
| Loop 4 | "As Appropriate" (Hiring Manager) |

## Microsoft Values
- **Growth Mindset** — Learn from failures, embrace challenges
- **Customer Obsession** — Empathy for users
- **Diversity & Inclusion** — Collaborative, inclusive thinking
- **One Microsoft** — Cross-team collaboration

## Common Behavioral Questions
- "Tell me about a time you failed and what you learned"
- "How do you handle disagreements in technical decisions?"
- "Describe a time you had to learn something new quickly"
- "How do you ensure your team delivers quality?"

## System Design Focus
- Azure-centric solutions appreciated (but not required)
- Design Teams/Outlook/OneDrive-scale systems
- Focus on reliability and enterprise features
- Multi-tenant architecture is a plus

## Preparation Checklist
- [ ] Prepare "growth mindset" stories (learning from failure)
- [ ] Review Azure services (App Service, Cosmos DB, Service Bus)
- [ ] Practice system design with enterprise focus
- [ ] Prepare collaborative leadership stories
