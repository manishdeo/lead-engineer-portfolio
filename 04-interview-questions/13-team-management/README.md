# 👥 Conflict Resolution & Team Management (40+ Questions)

---

### Q1: How do you handle a conflict between two senior engineers?
**A:** Listen to both sides separately → Facilitate joint discussion focused on data/trade-offs (not opinions) → If no consensus, make the call based on criteria → Document decision → "Disagree and commit."

### Q2: How do you give constructive feedback?
**A:** SBI model — Situation (when/where), Behavior (what you observed), Impact (effect on team/project). Be specific, timely, private for negative feedback. Balance with positive recognition.

### Q3: How do you handle underperformance?
**A:** Diagnose root cause (skill gap? motivation? personal?) → Set clear expectations → Provide support (training, mentoring, pair programming) → Regular check-ins → Document progress → Escalate if no improvement.

### Q4: How do you build a high-performing engineering team?
**A:** Hire for growth mindset, create psychological safety, set clear goals (OKRs), invest in developer experience (tooling, CI/CD), regular 1:1s, celebrate wins, blameless postmortems.

### Q5: How do you handle a team member who resists change?
**A:** Understand their concerns (often valid), involve them in the decision process, show data/evidence, start with small pilot, acknowledge the transition cost, give time to adapt.

### Q6: How do you run effective code reviews?
**A:** Focus on: correctness, readability, maintainability, security. Not on: style (use linters). Keep PRs small (< 400 lines). Review within 24 hours. Be kind — suggest, don't demand. Approve with minor comments.
